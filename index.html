Welcome to the Wear OS Tutorial

1.OverView

Our focus area is on Android Wear OS. We decide to make a business card Wear OS application. That completely runs on the android wear OS watch itself, it does not rely on any smartphone device at all

2.Getting Start

Setting up the environment

Go to the Android Studio official website(https://developer.android.com/studio/index.html) download the newest version of android studio and then installs it.

2. Use the SDK manager to confirm that you have the latest version

Creating a new Wear OS application

  1. Click File>New>New Project
  2. It should bring you to the Configuration screen, go the the “Wear OS” section and select “Blank Wear Activity” then click the blue “Next” button.

  1. On the "Target Android Devices screen, pick only the "Wear" Option, and for the minimum SDK pick the latest available option. Then hit the blue “Next” button

Note:A friendly reminder, developing an android application, the API level of android image matters. Too early version of android image may lack some library support, which may bring a headache during the developing process. Too late version of the API, it may limit your application only work for certain android wear devices that have some level or high level android systems. So you need to choose the API level properly.

     4. On the "ADD an Activity to Wear screen, pick "Blank Wear Activity and hit Next.

     5. On the "Customize the Activity" click "Finish" and boom android studio created a     new Wear OS app for you

  1. Now we have to make a few changes to the build.gradle In the build.gradle for the app module confirm the compile SDK is set to 26 confirm the target SDK is set to 26 Update the dependencies section add these lines

dependencies {

   implementation fileTree(dir: 'libs', include: ['*.jar'])

   implementation 'com.google.android.support:wearable:2.5.0'

   implementation 'com.google.android.gms:play-services-wearable:17.0.0'

   implementation 'androidx.percentlayout:percentlayout:1.0.0'

   implementation 'androidx.legacy:legacy-support-v4:1.0.0'

   implementation 'androidx.recyclerview:recyclerview:1.1.0'

   //noinspection GradleCompatible

   implementation 'com.android.support:wear:28.0.0'

   implementation 'androidx.appcompat:appcompat:1.0.2'

   implementation 'androidx.constraintlayout:constraintlayout:1.1.3'

   compileOnly 'com.google.android.wearable:wearable:2.5.0'

}

Set up the emulator and run the app

An Android Virtual Device (AVD) is a configuration that defines the characteristics of an Android phone, tablet, Wear OS, Android TV, or Automotive OS device that you want to simulate in the Android Emulator. The AVD Manager is an interface you can launch from Android Studio that helps you create and manage AVDs.

To open the AVD Manager, do one of the following:

  • Select Tools > AVD Manager icon like the

.

  • Click AVD Manager in the toolbar.

Then you will see a screen like this.

On the bottom left corner, you can see a “+ Create Virtual Device..” button, click on it.

It will bring you into this page. On the left Category section select ”Wear OS”. It will should up all the available wear os device options. You select the device you want to test your application, or the target device you about to write an application for it. Then click the blue “Next” button on the bottom right corner to go to next page.

It bring you into this page.After we select the target hardware device, we now need to select the image for the proper hardware. If you already download any of the image file, then you can just the blue “Next” button on the bottom.

Otherwise you will need to click the blue ”Download” button next to your selected image to download it. It looks like this. You just need to wait for it to download, then click the “Finish” button to bring back to previous picture display screen.

You should be seeing this screen, if previously you did correctly. You can name your Android Virtual Device for good management. Then click the blue “Finish” button to fully import the virtual device into your Android Studio.

Ok, you should back to the Android Studio, on the top select bar find the AVD dropdown menu, your android virtual device should be on the list, then select it.

Now you could just click the green arrow button that right next the the AVD dropdown menu to run the application. It will be performed on the chosen AVD.
